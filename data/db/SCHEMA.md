# Database Schema Reference

## db/papers.json

### Top-level structure
```json
{
  "metadata": { "last_updated": "YYYY-MM-DD", "owned_count": N, "stub_count": N },
  "papers": { "<paper_id>": { ... } }
}
```

### Paper ID convention
`{first_author_lastname}_{year}_{first_significant_title_word}` — all lowercase, underscores. Generated by agents, never by Python.

### Paper entry: `owned`
| Field | Type | Description |
|---|---|---|
| `id` | string | Canonical paper ID |
| `type` | `"owned"` | Has PDF + full extraction |
| `title` | string | Full paper title |
| `authors` | string[] | `"Lastname, Firstname"` format |
| `year` | int | Publication year |
| `journal` | string | Journal or venue |
| `doi` | string\|null | Normalized DOI (lowercase, no URL prefix) |
| `abstract` | string | Full abstract |
| `pdf_file` | string | Relative path in `data/pdfs/` |
| `text_file` | string | Relative path in `data/text/` |
| `extraction_file` | string | Path to rich extraction JSON |
| `cites` | string[] | IDs of papers this paper cites |
| `cited_by` | string[] | IDs of owned papers citing this |
| `aliases` | string[]? | IDs of earlier versions pointing here |
| `sections` | object[]? | Section headings + summaries (from Pass 4) |
| `extraction_meta` | object? | Extraction provenance metadata |

### Paper entry: `external_owned`
Imported from an external source (e.g. Semantic Scholar, manual import) without a local PDF. Treated as owned for all query, display, and metadata-counting purposes. Does NOT have `extraction_file` or `text_file`.

| Field | Type | Description |
|---|---|---|
| `id` | string | Canonical paper ID |
| `type` | `"external_owned"` | Owned externally — no local PDF/extraction |
| `title`, `authors`, `year`, `journal`, `doi`, `abstract` | as owned | |
| `cites` | string[] | Papers this paper cites (if known) |
| `cited_by` | string[] | IDs of owned papers citing this |
| `s2_paper_id` | string? | Semantic Scholar paper ID |

### Paper entry: `stub`
Referenced but no PDF. `cites` is always `[]` (unless discovered via S2 forward citations, in which case it lists the owned papers it cites).

| Field | Type | Description |
|---|---|---|
| `id`, `type`, `title`, `authors`, `year`, `journal`, `doi` | as above | |
| `cites` | string[] | Empty unless S2-discovered |
| `cited_by` | string[] | IDs of owned papers citing this |
| `superseded_by` | string? | Canonical ID if this is an earlier version |

### extraction_meta
| Field | Type | Description |
|---|---|---|
| `extractor` | string | Extractor name from `project.yaml` |
| `passes_completed` | int[] | Which passes ran (e.g. `[1, 2]`) |
| `detail_level` | string | `"metadata"` / `"contexts"` / `"analysis"` / `"full"` |
| `models` | object | Pass number → model name (e.g. `{"1": "haiku", "2": "sonnet"}`) |
| `agent_versions` | object | Agent name → git short SHA |
| `extracted_at` | string | ISO date of extraction |

### sections entry
| Field | Type | Description |
|---|---|---|
| `heading` | string | Section heading (e.g. "1. Introduction") |
| `summary` | string | Agent-generated section summary |

### cites vs cited_by
- `cites`: only populated for owned papers (and S2-discovered stubs)
- `cited_by`: populated for all paper types
- Bidirectional consistency enforced by `check_db.py`

## db/contexts.json

Citation contexts index — how each reference is used in each paper.

### Top-level structure
```json
{
  "generated": "YYYY-MM-DD",
  "owned_count": N,
  "total_papers": N,
  "owned_papers": [...],
  "by_cited": { "<paper_id>": [...] },
  "by_purpose": { "<purpose>": [...] }
}
```

### Context entry
| Field | Type | Description |
|---|---|---|
| `citing` | string | Paper ID doing the citing |
| `cited` | string | Paper ID being cited |
| `cited_title` | string | Title of cited paper |
| `purpose` | string | Why cited: `background`, `supporting_evidence`, etc. |
| `section` | string | Section of citing paper |
| `quote` | string | Exact passage containing the citation |
| `explanation` | string | Agent-generated explanation |

## db/authors.json

### Person entity
| Field | Type | Description |
|---|---|---|
| `id` | string | `lastname_firstname` all lowercase |
| `canonical_name` | string | Best known name form |
| `name_variants` | string[] | All name strings seen |
| `papers` | string[] | All paper IDs |
| `owned_papers` | string[] | Subset that are owned |
| `paper_count` | int | `len(papers)` |
| `coauthors` | string[] | Author IDs of co-authors |

### Institution entity
| Field | Type | Description |
|---|---|---|
| `id` | string | Normalized name |
| `name` | string | Original name |
| `type` | `"institutional"` | |
| `papers` | string[] | Paper IDs |

## project.yaml

Project configuration at repo root. Extraction settings:
- `extraction.extractor_name`: name stamped into extraction_meta
- `extraction.default_passes`: which passes run by default
- `extraction.default_model`: default model for extraction agents
- `extraction.pass_agents`: mapping of pass number → agent name

## Extraction JSON (data/extractions/{paper_id}.json)

After merge_extraction.py, contains all passes merged:
- Pass 1: `id`, `source_file`, `pdf_file`, `title`, `authors`, `year`, `journal`, `doi`, `abstract`, `citations[]`
- Each citation: `id`, `citation_key`, `authors`, `author_lastnames[]`, `year`, `title`, `title_normalized`, `journal`, `doi`, `contexts[]`
- Each context: `section`, `purpose`, `quote`, `explanation`
- Pass 3 (optional): `research_questions[]`, `methodology{}`, `claims[]`, `keywords[]`, `topics{}`
- Pass 4 (optional): `sections[]` with `heading`, `summary`, `annotated_text`
- `extraction_meta`: provenance block (extractor, passes, models, agent versions, date)
